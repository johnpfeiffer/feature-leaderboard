# Validation

This directory contains AI-derived proof obligations, tests, checks, and eval plans for the kernel.

The files here are not authoritative. If anything in this directory conflicts with `/KERNEL/`, the kernel wins.

## Files

- `acceptance-enumeration.md` - generated acceptance criteria from current kernel requirements.
- `traceability.md` - mapping from kernel items to derived spec and validation obligations.
- `test-plan.md` - recommended automated and manual validation strategy.

## Validation Rule

No implementation task is complete unless the relevant validation is run or explicitly deferred with a reason.
